# Framework & Integration 담당자 — 2차 빌드 개인 마일스톤

- **담당자:** 천성욱

## 1. 역할

업데이트된 프로젝트 기획서에 따라 Framework & Integration 영역의 2차 빌드 구현·통합·검증을 담당한다. 독립 기부 시스템은 범위에서 제외하고, 투자 퀘스트·구조 대출·멀티 Caravan·저장 안정성 정책을 반영한다.

## 2. 공통 일정 원칙

- **2차 빌드 개발 기간:** 2026-07-20 ~ 2026-08-06
- **최종 빌드 개발 기간:** 2026-08-07 ~ 2026-08-11
- **기준 브랜치:** `dev2`
- **통합 방식:** 기능 브랜치 → PR → `dev2`
- **구조 동결:** 2026-08-04 업무 종료
- **기능 동결:** 2026-08-05 업무 종료
- **2차 빌드 RC:** 2026-08-06
- **1차 빌드 고정:** 1차 빌드 범위와 통과 조건은 소급 수정하지 않는다.
- **비주얼 세부 정책:** 핵심 기능과 데이터 흐름 안정화 후 별도 합의로 확정한다.


## 3. 최종 책임 범위

- 담당 영역의 제품용 전체 플레이 흐름 완성
- 멀티 Caravan과 `caravanId + full tradeId` 계약 준수
- 투자 퀘스트·구조 대출·마차 파괴·수리·건물 업그레이드 관련 담당 기능
- 저장 실패 시 성공 상태가 확정되지 않도록 Framework transaction과 연동
- RC 이전 구조 동결과 전체 회귀 테스트

## 5. 핵심 기능 완료 후 비주얼 기획 협업

- 비주얼 구조 변경이 현재 담당 도메인의 저장·계산·런타임 계약을 깨지 않는지 검토한다.
- 거점 시설 선택형 준비 흐름과 미니맵 연동에 필요한 최소 데이터·Command·ViewData를 제안한다.
- 세부 구현은 핵심 기능 완료 및 공용 구조 동결 이후 별도 합의로 진행한다.

## 6. 완료 기준

- 제품 UI와 공용 API를 통해 담당 기능이 동작한다.
- 재실행 후 담당 상태와 효과가 일관되게 복구된다.
- 중요 Command 저장 실패 시 변경이 rollback된다.
- 독립 기부·누적 투자·이자 대출 등 폐기된 정책을 구현하지 않는다.
- 8월 5일 이후 신규 대형 기능을 추가하지 않는다.

## 7. 일정 지연 시 축소

- 연출과 콘텐츠 수량을 우선 축소한다.
- 데이터 구조·중복 보상 방지·저장 rollback·투자/대출 핵심 transaction은 축소하지 않는다.
- 비주얼 세부 구현은 후속 정책으로 이관할 수 있으나 방향성 문서는 유지한다.
