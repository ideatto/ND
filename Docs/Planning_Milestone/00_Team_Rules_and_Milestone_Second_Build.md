# 누워서 돈벌기 — 2차 빌드 팀 공통 마일스톤 및 운영 규칙
## 선택 A — 최종 기획 범위 유지형

## 1. 문서 목적

이 문서는 5인 팀이 2차 빌드까지 최종 기획의 핵심 기능을 대부분 구현하기 위한 공통 목표, 일정, 우선순위, 책임 경계와 통합 기준을 정의한다.

1차 빌드는 고정된 기준선으로 유지한다. 2차 빌드에서는 단순한 Vertical Slice를 넘어 플레이어용 핵심 루프, 성장·경제, 계절·재난, 이벤트, 기부·투자·대출, 콘텐츠 확장과 저장 구조까지 구현한다.

## 2. 2차 빌드 최종 목표

```text
Caravan 준비
→ 도시·무역로·상품·식량·마차·견인 동물·용병 선택
→ 출발 검증
→ Traveling
→ 계절·재난·로드 이벤트·약탈
→ 성공 또는 실패 정산
→ Claim
→ 플레이어·Caravan·거점 성장
→ 기부·투자·대출
→ 신규 지역 및 콘텐츠 해금
→ 다음 무역
```

### 완료 기준

- 플레이어가 Debug Harness 없이 전체 무역 루프를 완주한다.
- 성공·실패·오프라인 완료·SettlementPending 복구가 제품 UI에서 동작한다.
- 성장과 거점 건물 효과가 다음 무역에 반영된다.
- 계절·재난·거리·이벤트가 가격과 손실에 영향을 준다.
- 기부·투자·대출이 각각 최소 1회 완성된 플레이 흐름으로 동작한다.
- 신규 지역 또는 신규 무역로 1개를 투자로 해금한다.
- 최종 목표 콘텐츠의 70~80% 이상이 2차 빌드에 입력된다.
- SaveData 구조가 최종 빌드 전에 동결된다.
- 최종 기간에 SaveData와 공용 API의 구조 변경이 필요하지 않다.

## 7. 거점 도시 UX 확장 가능성

현재 마일스톤은 거점 건물의 비용·효과·성장 표시를 최소 범위로 포함한다.

추후 별도 기획이 확정되면 다음과 같은 거점 도시 UX 확장 작업이 추가될 수 있다.

- 건물 배치·선택·정보 표시 개선
- 도시 성장 단계의 시각적 변화
- 건물별 상세 패널과 상호작용
- 거점 활동과 무역 준비 화면 연결
- 도시 상태·해금·생산 정보의 가시화

이 확장 항목은 현재 확정된 2차 필수 범위로 간주하지 않으며, **Content & Tools와 UI & Data가 공동으로 기획·제작·통합하는 후속 작업 후보**로 둔다.
