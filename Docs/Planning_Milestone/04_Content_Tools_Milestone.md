# Content & Tools 담당자 — 1차 빌드 개인 마일스톤

## 1. 역할

도시·상품·무역로·이벤트·마차·견인 동물·용병 콘텐츠와 데이터 검증, 디버그 도구, 테스트 시나리오, 밸런스 자료를 담당한다.

**최종 마감:** 2026년 7월 16일 오전 10시  
**기능 동결:** 2026년 7월 14일 오후 6시  
**문서 업데이트:** 2026년 7월 9일 — 마차 슬롯·적정 적재량·견인 동물 추가 효율 적재량·로드 이벤트 대응 데이터 반영

---

- 테스트 도시·상품·무역로
- 특산품

- 견인 동물 이동속도·식량 소모·추가 효율 적재량·구매 가격 데이터
- 용병 데이터
- 긍정·부정·약탈·로드 이벤트
- 마차별 로드 이벤트 대응 ID/태그와 긍정 효과 보정값
- 계절·재난 보정표
- 기부·투자·대출 콘텐츠 값
- 숨겨진 상품과 미개척지
- 아이콘·설명·결과 문구
- 튜토리얼·마스코트 대사
- 중복 ID 검사
- 누락 참조 검사
- 테스트 프리셋

- 런타임 상태 로직

- 전체 UI 레이아웃

# 3. 마일스톤

## M0 — 7월 7일 오후 8시

### 해야 할 일

- 거점 도시 1개
- 무역 도시 3개
- 무역로 3개 이상
- 상품 5개
- 특산품 1개
- 마차 2개: 슬롯 수·슬롯 중첩·기본 적정 적재량·최대 적재량·내구도·최소 동물 수 포함
- 견인 동물 2개: 기본 이동속도·식량 소모·추가 효율 적재량·구매 가격 포함
- 견인 동물 성장·등급 필드는 생성하지 않음
- 용병 2개
- 데이터 생성 규칙
- 중복 ID 검사
- 누락 참조 검사
- 마차 로드 이벤트 대응 ID/태그 유효성 검사
- 기본 적정 적재량 ≤ 최대 적재량 검사
- 데이터 초기화 도구

### 완료 기준

- 모든 테스트 데이터가 유효한 ID를 가진다.
- 도시에서 참조하는 상품과 무역로가 존재한다.
- 코드 수정 없이 데이터 교체가 가능하다.
- 잘못된 참조를 실행 전 또는 에디터에서 검출한다.

### 필요한 협업

- UI & Data: 데이터 필드와 아이콘 규격
- Core: 마차·동물·용병 필수 필드
- Progression: 가격과 보정 필드
- Framework: 데이터 초기화·디버그 연결

## M1 — 7월 9일 오후 10시

### 해야 할 일

- 도시별 판매 상품 구성
- 도시별 특산품 배치
- 기본 가격 입력
- 상품 아이콘과 설명
- 정상 무역 프리셋
- 슬롯 중첩 경계 프리셋
- 슬롯 수 초과 프리셋
- 적정 적재 프리셋
- 과적 프리셋
- 최대 적재량 초과 프리셋
- 견인 동물 추가 효율 적재량 비교 프리셋
- 서로 다른 견인 동물 혼합 오류 프리셋
- 빈 식량 프리셋
- 정산 기본 문구
- 성장 테스트 값

### 완료 기준

- 3개 도시의 상품 구성이 다르다.
- 한 도시에서 구매하고 다른 도시에서 판매 가능하다.
- 정상·오류 테스트 데이터를 빠르게 불러올 수 있다.
- 적정 적재·과적·적재 불가와 슬롯 경계를 각각 재현할 수 있다.
- UI에서 누락 아이콘과 설명이 없다.

## M2 — 7월 11일 오후 10시

### 해야 할 일

- 긍정 로드 이벤트 1개
- 부정 로드 이벤트 1개
- 각 이벤트에 대응하는 마차 데이터 1개 이상
- 약탈 이벤트 1개
- 거리별 이벤트 테이블
- 여름·겨울 보정표
- 가뭄·홍수 보정표
- 마차 슬롯·적정/최대 적재량·이벤트 대응 수치
- 견인 동물 이동속도·식량 소모·추가 효율 적재량 수치
- 용병·몬스터 전투력
- 단거리·장거리 프리셋
- 약탈 성공·실패 프리셋
- 경제 비교 시트
- 이벤트 강제 발생 데이터

### 완료 기준

- 모든 이벤트에 결과 문구와 정산 항목이 있다.
- 마차 대응 부정 이벤트는 무효화되고 긍정 이벤트는 강화되는 테스트 데이터가 있다.
- 견인 동물 추가 효율 적재량이 최대 적재량을 증가시키지 않는다.
- 계절·재난별 가격 차이를 테스트할 수 있다.
- 용병 유무에 따른 약탈 결과를 비교할 수 있다.
- 극단적인 음수·무한 수익 데이터가 없다.

### 일정 지연 시 축소

- 이벤트를 각각 1개로 고정
- 재난은 가격 보정만 입력
- 용병 등급과 몬스터 타입을 2개 이하로 제한
- 과공급 데이터는 후순위

## M3 — 7월 13일 오후 10시

### 해야 할 일

- 기부 단계 1개
- 기부 효과 1개
- 숨겨진 상품 1개
- 미개척지 1개
- 해금 도시 또는 무역로 1개
- 투자 요구량
- 대출 원금·상환 테스트 값
- 파산 테스트 프리셋
- 저장 회귀 테스트 체크리스트
- 신규 게임 초기 데이터

### 완료 기준

- 기부 후 상품 또는 위험도가 변한다.
- 투자 완료 후 신규 콘텐츠가 열린다.
- 대출 후 최소 무역이 가능하다.
- 저장 전후 데이터 비교 시 누락이 없다.

### 일정 지연 시 축소

- 투자 요구량을 재화 하나로 제한
- 기부 단계 1개만 사용
- 대출 값을 고정
- 해금 콘텐츠를 무역로 1개로 제한

# 4. 데이터 검증 체크리스트

- [ ] 모든 데이터에 고유 ID가 있다.
- [ ] 참조 대상 ID가 존재한다.
- [ ] 가격과 수량이 음수가 아니다.
- [ ] 마차 최소 동물 수가 데이터와 일치한다.
- [ ] 마차 기본 적정 적재량이 최대 적재량 이하이다.
- [ ] 슬롯 최대 개수와 슬롯별 최대 중첩 수량이 0 이상이다.
- [ ] 견인 동물 추가 효율 적재량과 식량 소모량이 음수가 아니다.
- [ ] 견인 동물 성장·등급 필드는 1차 빌드 필수 데이터에 포함되지 않는다.
- [ ] 마차 로드 이벤트 대응 ID 또는 태그가 실제 이벤트 데이터와 연결된다.
- [ ] 이벤트 결과가 정산 항목과 연결된다.
- [ ] 숨겨진 상품의 해금 조건이 존재한다.
- [ ] 투자 대상이 잠금 상태로 시작한다.
- [ ] 대출 후 최소 무역이 가능하다.
- [ ] 튜토리얼 데이터로 진행 불가가 발생하지 않는다.
- [ ] 빌드에 필요한 아이콘과 리소스가 포함된다.

---

# 5. 개인 완료 체크리스트

- [ ] 최소 콘텐츠 수량을 충족한다.
- [ ] 코드 수정 없이 값을 조정할 수 있다.
- [ ] 정상·실패·슬롯·적재·과적·이벤트 대응 경계값 프리셋이 있다.
- [ ] 데이터 검사 결과가 팀에 공유된다.
- [ ] 기능 동결 이후 변경 내역을 기록한다.
- [ ] RC 빌드의 데이터 버전을 식별할 수 있다.
