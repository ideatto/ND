# UI & Data 담당자 — 1차 빌드 개인 마일스톤

## 1. 역할

공용 데이터 정의, Caravan 준비 화면, 슬롯·적재·견인 동물 정보, 진행 화면, 정산, 성장, 기부·투자·대출, 튜토리얼과 정보 전달 UX를 담당한다.

**최종 마감:** 2026년 7월 16일 오전 10시  
**기능 동결:** 2026년 7월 14일 오후 6시  
**문서 업데이트:** 2026년 7월 9일 — Caravan 슬롯·적정 적재량·과적·견인 동물·시간 기반 식량 표시 반영

- 공용 데이터 구조 초안
- 데이터 ID와 표시 데이터 구분
- 도시·무역로 선택
- 상품 구매·판매
- 식량·마차·동물·용병 선택
- 슬롯 사용량·중첩 규칙 표시
- 기본·추가·최종 적정 적재량과 최대 적재량 표시
- 과적 상태·과적 비율·예상 속도 감소 표시
- 견인 동물 종류·수·식량 소모·추가 효율 적재량 표시
- 출발 조건 표시
- 무역 진행률
- 기본·상세 정산
- 과적 지연·총 식량 소모·마차 로드 이벤트 대응 결과 표시
- 성장 화면
- 계절·재난·위험·실패 원인 표시

# 3. 마일스톤

## M0 — 7월 7일 오후 8시

### 해야 할 일

- `TradeItemData`, `TownData`, `RouteData`, `WagonData`, `DraftAnimalData`, `RoadEventData`, `TradeResultData`, `SaveData` 초안
- 슬롯·적정 적재량·과적 표시용 View Data 계약
- ID와 표시명 분리
- UI View Data 구조
- 화면 흐름 와이어프레임
- 공용 선택 리스트·정보 패널 설계
- 더미 도시와 상품 표시
- 오류와 상태 메시지 형식 정의

### 완료 기준

- 더미 데이터가 화면에 표시된다.
- 데이터 ID를 UI 텍스트로 직접 사용하지 않는다.
- Core와 Progression 계산 결과를 받을 View Data가 정의된다.
- 공용 데이터 변경 사항을 팀에 공유한다.

### 필요한 협업

- Core: 출발 불가 원인과 Caravan 표시 데이터
- Core: 슬롯 사용량·최종 적정 적재량·과적 비율·이동속도 보정
- Framework: 화면 전환과 저장 상태
- Progression: 가격·성장 표시 데이터
- Content: 아이콘·설명·표시 문구

## M1 — 7월 9일 오후 10시

### 해야 할 일

- 도시 선택
- 무역로 선택
- 상품 구매·판매
- 식량 적재
- 마차·견인 동물·용병 선택
- 한 종류의 견인 동물만 선택하도록 입력 제한
- 현재 슬롯 수·최대 슬롯 수 표시
- 아이템 중첩 가능 여부·최대 중첩 수량 표시
- 현재 적재량·기본 적정 적재량·추가 효율 적재량 표시
- 최종 적정 적재량·최대 적재량 표시
- 과적 상태·과적 비율·예상 속도 감소 표시
- 과적을 반영한 예상 이동 시간
- 출발 불가 원인
- 출발 버튼 상태
- 진행률
- 기본 정산
- 성장 구매

### 완료 기준

- 코드 로그 없이 무역을 준비한다.
- 슬롯 초과, 최대 적재량 초과, 견인 동물 혼합 등 잘못된 구성은 버튼 비활성화 또는 경고로 확인된다.
- 적정 적재량 초과이면서 최대 적재량 이하인 구성은 과적 경고와 함께 출발할 수 있다.
- 정산에서 구매 비용, 판매 수익, 순이익을 확인한다.
- 성장 구매 후 변경 수치를 확인한다.

### 일정 지연 시 축소

- 각 화면을 별도 씬으로 분리하지 않는다.
- 공용 리스트와 상세 패널을 재사용한다.
- 애니메이션과 전환 연출을 최소화한다.

### 해야 할 일

- 저장 결과 또는 오류 피드백
- 성장 전후 비교

- 기부 UI

- 신규 지역 해금
- 대출 원금·상환 상태

- 두 재화의 사용처가 혼동되지 않는다.
- 해금된 지역과 상품을 즉시 확인한다.

### 일정 지연 시 축소

- 기부·투자·대출을 공용 패널로 구현
- 기여도와 발전도를 하나의 게이지로 표시
- 복잡한 팝업 대신 결과 텍스트와 핵심 수치 사용

# 5. 개인 완료 체크리스트

- [ ] 준비 화면만으로 무역 출발이 가능하다.
- [ ] 슬롯·적재·과적·견인 동물 정보를 코드 로그 없이 이해할 수 있다.
- [ ] 견인 동물 성장·등급을 확정 정보처럼 표시하지 않는다.
- [ ] 마차의 로드 이벤트 무효화·강화 결과를 정산에서 확인할 수 있다.
- [ ] 정산에서 이익·손실 이유를 이해할 수 있다.
- [ ] UI가 계산을 중복 수행하지 않는다.
- [ ] 저장·이어하기 상태를 명확히 표시한다.
- [ ] 튜토리얼과 스킵이 모두 동작한다.
- [ ] 핵심 화면이 목표 해상도에서 깨지지 않는다.
