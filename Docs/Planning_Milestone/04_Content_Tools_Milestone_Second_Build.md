## 선택 A — 최종 기획 범위 유지형

## 1. 역할

도시·상품·무역로·마차·동물·용병·계절·재난·이벤트·기부·투자·대출 콘텐츠와 데이터 검증 도구를 담당한다.

## 공통 일정 원칙

- **2차 빌드 개발 기간:** 2026-07-20 ~ 2026-08-06
- **2차 빌드 작업일:** 14일
- **최종 빌드 개발 기간:** 2026-08-07 ~ 2026-08-11
- **최종 빌드 작업일:** 3일
- **주말 및 공휴일:** 작업을 배정하지 않는다.
- **1차 빌드 기획:** 2026-07-17 확정본을 고정하며 소급 수정하지 않는다.
- **기준 브랜치:** `dev2`
- **통합 방식:** 기능 브랜치에서 작업하고 PR을 통해 `dev2`에 반영한다.
- **2차 빌드 목표:** 최종 기획 기능의 약 90%를 구현하고, 최종 빌드 기간에는 신규 대형 시스템 개발을 금지한다.
- **구조 동결:** 2026-08-04 업무 종료 시점
- **기능 동결:** 2026-08-05 업무 종료 시점
- **2차 빌드 RC:** 2026-08-06
- **최종 빌드 기간:** 회귀 테스트, Blocker 수정, 밸런스, 콘텐츠 누락 보완, 연출, 제출 안정화만 수행한다.

## 4. 거점 도시 UX 확장 가능성

추후 거점 도시 UX 확장 기획이 확정되면 UI & Data와 공동으로 다음 콘텐츠 제작을 맡을 수 있다.

- 건물별 상세 설명·아이콘·상태 문구
- 성장 단계별 도시·건물 시각 리소스
- 해금·생산·효과 데이터
- 거점 상호작용용 콘텐츠 프리셋

현재 마일스톤에서는 최소 건물 비용·효과와 표시 리소스만 필수로 한다.

## 5. 완료 기준

- 최종 기간에는 신규 시스템용 데이터 구조를 추가하지 않는다.
- 2차 RC에 최종 목표 콘텐츠의 대부분이 포함된다.
- 모든 데이터가 유효한 ID와 참조를 가진다.
