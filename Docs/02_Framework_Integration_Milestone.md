# Framework & Integration 담당자 — 1차 빌드 개인 마일스톤

## 1. 역할

씬, 공용 매니저, 시간, 저장, 오프라인 진행, 시스템 간 이벤트 전달, 통합 빌드와 릴리스 후보를 담당한다.

**최종 마감:** 2026년 7월 16일 오전 10시  
**기능 동결:** 2026년 7월 14일 오후 6시

# 2. 최종 책임 범위

- Boot → Title → Loading → InGame → Title
- 공용 매니저 생명주기
- 게임 시간 서비스
- 씬 전환과 중복 입력 방지

- 자동 저장
- 오프라인 경과 시간

# 3. 마일스톤

## M0 — 7월 7일 오후 8시

### 해야 할 일

- Boot, Title, Loading, InGame 씬 연결
- 공용 매니저 중복 생성 방지
- 새 게임과 이어하기 분기 인터페이스
- 게임 시간 서비스 초안
- 저장 서비스 인터페이스
- 저장 데이터 버전 필드
- 공용 로그
- 시간 배속
- 무역 즉시 완료 요청 이벤트
- 통합용 샘플 씬

### 완료 기준

- 씬 흐름이 처음부터 끝까지 동작한다.
- 공용 매니저가 중복 생성되지 않는다.
- 더미 데이터 로드 완료 이벤트가 발생한다.
- 각 담당자가 시간과 저장 API를 호출할 수 있다.

### 필요한 협업

- UI & Data: `SaveData` 정의
- Core Gameplay: 저장해야 할 상단 상태
- Progression: 계절·재난 시간 요구사항
- Content: 테스트 데이터 초기화

## M1 — 7월 9일 오후 10시

### 해야 할 일

- 무역 시작 시 게임 시간 기록
- 종료 예정 시간 계산 연결
- 무역 완료 이벤트
- InGame 내부 화면 상태 연결
- 정산 화면 진입
- 정산 종료 후 준비 상태 전환
- 새 게임 기본 데이터 초기화
- `Dev2` 브랜치에서 매일 통합 빌드

### 완료 기준

- 무역 출발 후 정상적으로 완료된다.
- 화면 상태와 런타임 상태가 어긋나지 않는다.
- 정산 종료 후 다음 무역으로 복귀한다.
- `Dev2` 브랜치가 실행 가능한 상태를 유지한다.

### 일정 지연 시 축소

- LoadingScene 연출 최소화
- 화면 전환은 단일 라우터로 제한
- 복잡한 서비스 로케이터 또는 DI 도입 금지

- 거리 이벤트 발생 시점

- 같은 무역 ID에서 완료 이벤트가 한 번만 발생한다.

- 시간 조작 방어는 역행 감지와 상한만 유지

# 5. 통합 체크리스트

- [ ] `Dev2` 브랜치가 매일 빌드된다.
- [ ] 공용 매니저가 중복되지 않는다.
- [ ] 씬 이동 중 입력이 중복 처리되지 않는다.
- [ ] 동일 무역 보상이 한 번만 지급된다.
- [ ] 저장 실패가 로그와 UI에 전달된다.
- [ ] 잘못된 저장 데이터에서 안전하게 새 게임으로 전환한다.
- [ ] 제출용 빌드와 소스 커밋을 연결할 수 있다.
