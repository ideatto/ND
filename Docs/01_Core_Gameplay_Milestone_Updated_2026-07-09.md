# Core Gameplay 담당자 — 1차 빌드 개인 마일스톤

## 1. 역할

Caravan 구성, 슬롯과 적재 검증, 무역 실행 상태, 시간 기반 식량 소모, 마차, 견인 동물, 용병, 로드 이벤트 결과 적용, 성공·실패 흐름을 담당한다.

**최종 마감:** 2026년 7월 16일 오전 10시  
**기능 동결:** 2026년 7월 14일 오후 6시  
**문서 업데이트:** 2026년 7월 9일 — Caravan 슬롯·적정 적재량·과적·견인 동물·로드 이벤트·시간 기반 식량 반영

---

# 2. 최종 책임 범위

- Caravan 구성과 출발 조건
- 슬롯 사용량과 중첩 계산
- 현재 적재량·최종 적정 적재량·과적 비율 계산
- 무역 상태 전환
- 이동 진행과 완료
- 경과 인게임 시간 기반 식량 소모와 부족
- 마차 내구도
- 견인 동물 최소 조건과 단일 종류 할당 검증
- 견인 동물 추가 효율 적재량 합산
- 용병 배치와 계약
- 약탈 결과 적용
- 마차의 로드 이벤트 부정 효과 무효화·긍정 효과 강화
- 상품·내구도·용병 손실
- 무역 실패와 거점 복귀
- 성장 스탯의 런타임 반영
- 튜토리얼 전용 실패 방지
- 이전 무역 구성 재적용

담당하지 않는 항목:

- JSON 파일 직접 저장
- 경제 수치 정책 최종 결정
- UI 오브젝트 직접 구성
- 콘텐츠 데이터 입력과 문구 제작

# 3. 마일스톤

## M0 — 7월 7일 오후 8시

### 해야 할 일

- `CaravanData`, `WagonData`, `DraftAnimalData` 필드 요구사항 확정
- 마차 슬롯 정의와 중첩 규칙 계약 확정
- 마차 기본 적정 적재량·최대 적재량 계약 확정
- 견인 동물 추가 효율 적재량 계약 확정
- 견인 동물 성장·등급은 1차 빌드 계약에서 제외
- 무역 상태 enum 협의
- 출발 조건 목록 확정
- 로드 이벤트 ID/태그와 긍정·부정 구분 입력 구조 확정
- 이벤트 결과 입력 구조 확정
- 정산 결과로 전달할 손실 항목 확정
- 더미 Caravan 구성 생성
- 테스트 코드로 출발 가능 여부 확인

### 산출물

- Caravan 상태 정의
- 출발 검증 API
- 무역 시작·완료 인터페이스
- UI 없이 실행 가능한 더미 테스트

### 완료 기준

- 상품, 식량, 마차, 견인 동물, 용병 데이터를 한 Caravan에 구성할 수 있다.
- 동일 아이템의 중첩 수량으로 필요한 슬롯 수를 계산할 수 있다.
- 최종 적정 적재량이 최대 적재량을 초과하지 않는다.
- 누락 구성과 적재량 초과를 코드에서 판별한다.
- Framework 담당자에게 저장 가능한 상태를 설명할 수 있다.

### 필요한 협업

- UI & Data: `CaravanData`, 출발 불가 코드
- Framework: 시간과 상태 전환
- Progression: 적재량·속도·식량 계산 규칙
- Content: 더미 마차·견인 동물·용병 데이터

---

## M1 — 7월 9일 오후 10시

### 해야 할 일

- 상품과 식량 적재
- 아이템 슬롯 사용량과 중첩 수량 계산
- 마차, 견인 동물, 용병 선택 반영
- 한 종류의 견인 동물만 할당하도록 검증
- 마차 기본 적정 적재량과 최대 적재량 적용
- 견인 동물 수에 따른 전체 추가 효율 적재량 계산
- 최종 적정 적재량 계산
- 현재 적재량·과적 상태·과적 비율 계산
- 출발 가능 조건 검사
- 무역 시작
- 진행 상태
- 정상 도착
- 상품 판매 결과 전달
- 정산 후 재출발 상태 복귀
- 성장 효과 1종 이상 반영

### 산출물

```text
Prepare
→ Ready
→ Traveling
→ Arrived
→ SettlementPending
→ Completed
→ Prepare
```

### 완료 기준

- 무역 한 사이클을 3회 반복한다.
- 슬롯 수 또는 최대 적재량을 초과하면 출발하지 않는다.
- 적정 적재량을 초과하지만 최대 적재량 이하인 경우에는 과적 상태로 출발할 수 있다.
- 서로 다른 종류의 견인 동물을 혼합하면 출발하지 않는다.
- 도착 결과가 `TradeResultData`로 전달된다.
- 성장 후 적재량 또는 속도가 달라진다.

### 일정 지연 시 축소

- 용병은 1종 고정
- 견인 동물은 1종 고정
- 복잡한 상태 머신 대신 명확한 enum 기반 상태 전환 사용

## M2 — 7월 11일 오후 10시

### 해야 할 일

- 거리별 이동 시간 적용
- 과적 비율에 따른 이동속도 보정 적용
- 로드 이벤트에 따른 이동속도 변경 적용
- 경과 인게임 시간 기반 식량 예상량과 실제 소모
- 식량 부족 시 속도 저하
- 식량 부족 지속 시 실패
- 마차 내구도 감소
- 최소 견인 동물 수 검사
- 거리별 이벤트 발생 요청
- 대응 가능한 부정 로드 이벤트 효과 무효화
- 대응 가능한 긍정 로드 이벤트 효과 강화
- 약탈 결과 적용
- 상품과 내구도 손실
- 실패 후 거점 복귀
- 손실 상한 적용

### 완료 기준

- 식량을 적게 가져가면 실패할 수 있다.
- 마차·견인 동물 선택이 속도 또는 적재량에 영향을 준다.
- 용병 유무가 약탈 손실에 영향을 준다.
- 실패 후 다시 무역을 시작할 수 있다.
- 모든 결과가 정산 데이터로 전달된다.
- 실제 이동시간, 총 식량 소모, 출발 적재량, 최종 적정 적재량, 과적 비율과 마차 이벤트 대응 결과가 정산 데이터에 포함된다.

### 필요한 협업

- Progression: 식량 공식, 전투력 비율, 손실 상한
- Framework: 이벤트 시점, 경과 인게임 시간, 시간 배율
- UI: 예상량·위험도·실패 원인
- Content: 이벤트와 전투력 데이터

### 일정 지연 시 축소

- 용병 소멸 제외
- 견인 동물 손실 제외
- 약탈 결과를 상품 손실과 내구도 손실만으로 제한

## M3 — 7월 13일 오후 10시

### 해야 할 일

- 런타임 Caravan 원본 상태를 저장용 DTO로 제공
- 마차 ID·현재 내구도·견인 동물 ID와 수·화물·식량·발생 이벤트 결과를 저장 대상으로 제공
- 최종 적정 적재량·과적 비율·최종 이동속도는 저장하지 않고 복구 후 재계산
- 재접속 후 진행 중·완료 상태 복구
- 오프라인 완료 후 정산 대기 전환
- 성장 효과 재적용
- 기부 효과가 위험도 또는 이벤트에 반영되도록 연결
- 신규 무역로 해금 후 선택 가능 처리
- 대출 후 최소 구성으로 출발 가능 처리

### 완료 기준

- 무역 중 종료 후 재실행해 상태가 복구된다.
- 완료 보상이 두 번 발생하지 않는다.
- 저장된 성장 효과가 유지된다.
- 파산 구제 후 다시 출발할 수 있다.

### 일정 지연 시 축소

- 기부 효과는 위험도 감소 1개
- 투자 해금은 무역로 1개
- 대출은 최소 자금 지급만 연결

- 이전 구성을 한 번에 적용할 수 있다.
- UI가 필요한 모든 계산값을 API로 받는다.
