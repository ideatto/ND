# Progression & System 담당자 — 1차 빌드 개인 마일스톤

## 1. 역할

구매·판매 가격, 성장, 계절, 재난, 거리 보상, 과적 이동속도 보정, 식량 소비 밸런스, 재화 이원화, 기부, 투자, 파산과 대출 규칙을 담당한다.

**최종 마감:** 2026년 7월 16일 오전 10시  
**기능 동결:** 2026년 7월 14일 오후 6시  
**문서 업데이트:** 2026년 7월 9일 — 과적 속도 보정·최소 속도·견인 동물 수·시간 기반 식량 밸런스 반영

# 2. 최종 책임 범위

- 기본 구매·판매 가격
- 가격 보정 적용 순서
- 거리 보상
- 계절·재난 가격·수량 보정
- 과공급 여부
- 식량·용병·마차 비용 정책
- 과적 비율에 따른 이동속도 감소 공식과 최소 이동속도
- 견인 동물 수 보정과 시간당 식량 소모 밸런스
- 로드 이벤트 긍정 효과 강화 보정 범위
- 플레이어·Caravan 성장
- 견인 동물 성장·등급은 비확정 사항으로 1차 빌드 성장 계약에서 제외
- 성장 비용 증가
- 무역용 재화와 발전용 재화
- 기부 조건과 효과
- 투자 조건과 해금
- 파산 판정
- 대출 원금과 상환
- 손실 상한과 복구 가능성

담당하지 않는 항목:

- UI 오브젝트 직접 변경
- 저장 파일 직접 쓰기
- 무역 상태 전환
- 콘텐츠 데이터 대량 입력

---

# 3. 가격 계산 공통 순서

```text
기본 가격
→ 도시 기본 보정
→ 계절 보정
→ 재난 보정
→ 도시·무역로 이벤트 보정
→ 과공급 보정
→ 플레이어 구매 할인·판매 할증
```

모든 보정 항목은 정산 화면에 전달할 수 있어야 한다.

- 최소 이동속도 정책
- 견인 동물 수 보정 입력·출력 계약
- 단위 인게임 시간당 식량 소모 입력·출력 계약
- 식량·용병·마차 비용 항목 정의
- 계절·재난 데이터 요구사항
- 기부·투자·대출 상태 정의
- 손익 정산 항목 협의

### 완료 기준

- 같은 입력으로 같은 가격 결과를 낸다.
- 보정 항목별 결과를 분리해 반환한다.
- UI와 Core가 필요한 스탯을 명확히 알 수 있다.
- 데이터 값과 계산 코드를 분리한다.

### 필요한 협업

- UI & Data: 보정 항목 표시 방식
- Core: 성장 스탯과 실패 손실 적용
- Framework: 시간 기반 계절·재난·상환
- Content: 보정표와 비용 데이터
- Content: 마차 적정/최대 적재량과 견인 동물 추가 효율 적재량 기본값

## M1 — 7월 9일 오후 10시

### 해야 할 일

- 기본 구매 가격
- 기본 판매 가격
- 기본 판매 차익
- 식량 비용
- 용병 고용비
- 기본 정산
- 적정 적재 구간의 적재량 보정값 `1` 확정
- 과적 속도 보정 공식 초안
- 최소 이동속도 초안
- 성장 비용
- 플레이어 성장 1종
- Caravan 성장 1종
- 해당 성장 1종은 견인 동물 성장·등급을 요구하지 않는 항목으로 제한
- 발전용 재화 최소 획득과 사용

### 완료 기준

- 구매와 판매 후 순이익이 계산된다.
- 성장 구매 비용이 차감된다.
- 성장 효과가 Core 런타임에 반영된다.
- 정산 항목에 모든 비용이 분리된다.
- 과적 비율 입력으로 동일한 속도 보정값을 반환한다.
- 최소 이동속도 아래로 감소하지 않는다.

### 일정 지연 시 축소

- 플레이어 성장 1개
- Caravan 성장 1개
- 성장 비용은 단순 단계 증가
- 발전용 재화 사용처는 1개

## M2 — 7월 11일 오후 10시

### 해야 할 일

- 여름·겨울
- 가뭄·홍수
- 계절 가격 보정
- 재난 가격 또는 수량 보정
- 거리 보상
- 과적 비율별 이동속도 감소 곡선 확정
- 견인 동물 수 보정 확정
- 경과 인게임 시간·견인 동물 수·용병 수 기반 식량 소비 보정 확정
- 로드 이벤트 긍정 효과 보정 상한 확정
- 전투력 비율 규칙
- 실패 손실 상한
- 마차 내구도 비용 의미
- 재화 이원화
- 과공급 최소 로직 검토

### 완료 기준

- 같은 상품의 가격이 조건에 따라 달라진다.
- 단거리와 장거리의 기대 보상이 다르다.
- 적정 적재·중간 과적·최대 적재 경계에서 이동속도 보정이 단조롭게 감소한다.
- 견인 동물 추가로 얻는 적재 효율과 추가 식량 비용이 비교 가능하다.
- 용병 전투력 차이가 결과에 영향을 준다.
- 실패가 회복 불가능한 상태를 만들지 않는다.
- 두 재화의 사용처가 겹치지 않는다.

### 일정 지연 시 축소

- 과공급 제외 가능
- 재난은 가격 변화만 적용
- 거리 보상은 3단계 고정값
- 과적 속도 보정은 선형 공식 1개
- 견인 동물 수 보정은 고정 단계값
- 전투 공식은 전투력 비율 하나로 제한

### 해야 할 일

- 성장 비용 증가
- 플레이어·Caravan 성장 저장 값 정의
- 견인 동물 성장·등급 저장 값은 정의하지 않음
- 기부금과 기부 한도
- 기부 단계 1개
- 기부 효과 1개
- 숨겨진 상품 해금
- 투자 비용과 진행도
- 신규 지역 해금
- 파산 조건
- 대출 1종
- 추가 대출 제한
- 정산 시 자동 상환 또는 수익 차감
- 최소 무역 자금 보장

### 완료 기준

- 성장 후 경제 또는 Caravan 성능이 달라진다.
- 1차 빌드 성장은 견인 동물 성장·등급 데이터 없이 동작한다.
- 기부가 실제 수치 또는 상품에 영향을 준다.
- 투자 완료 후 새로운 지역이 열린다.
- 파산 후 대출로 다시 무역할 수 있다.
- 대출을 반복해 무한 자금을 얻을 수 없다.

### 일정 지연 시 축소

- 기여도와 발전도 통합
- 투자 1회 결제로 즉시 해금
- 대출 원금과 상환 비율 고정
- 기부 효과는 위험도 감소 또는 상품 해금 하나만 사용

# 6. 개인 완료 체크리스트

- [ ] 가격 계산 순서가 고정되어 있다.
- [ ] 보정 항목을 개별 표시할 수 있다.

- [ ] 식량 소모 공식은 경과 인게임 시간을 입력으로 사용한다.
- [ ] 실패 후 복구 가능한 경제 구조다.
- [ ] 기부·투자·대출의 목적이 서로 다르다.
