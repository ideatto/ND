- 최종 적정 적재량과 과적 판정

## 1차 빌드 제외

- 성장 스탯을 플레이어 루프에 반영
- 용병·약탈 **전투** 및 전투 실패로 인한 손실 적용
- 로드 이벤트 전체
- 계절·재난
- 기부·투자·대출 연결
- 튜토리얼 전용 로직
- 여러 Caravan

Traveling 중 내구도 0 → `WagonBroken` 실패는 **전투 이벤트 실패로 내구도가 깎일 때**를 전제로 한다. 전투는 미구현이므로 1차에서는 일반 거리 마모만으로 Traveling 실패를 내지 않도록 정리한다.

# 3. 현재 상태

## 검증 완료

- 정상 출발과 3회 반복 루프
- 최대 적재량 초과 차단
- 과적 출발 허용
- 견인 동물 혼합 차단
- 최소 동물 수 미달 차단
- 식량 감소와 식량 부족 지속 후 실패
- 성공·실패 정산 후 재출발

## 결함·확인 사항

- 식량 부족 실패가 `WagonBroken`으로 표시될 가능성
- 전투 미구현인데 내구도/`WagonBroken`이 일반·식량 실패와 섞일 가능성
- 식량 부족 속도 저하 로직과 On/Off 플래그 상태 확인
- 실패 결과가 UI까지 전달되는 매핑 확인 필요
- DraftAnimalData의 최대 적재량 증가 필드는 현재 규칙과 충돌

# 4. 작업 일정

## 2026-07-14 — UI 연결 계약 고정

- Preparation UI가 사용할 Caravan View Data 최종 확인
- 출발 검증 결과 코드와 표시용 메시지 키 정리
- 현재 슬롯, 적재량, 적정 적재량, 최대 적재량, 과적 여부 제공
- 견인 동물 종류·수와 검증 결과 제공
- UI 출발 요청이 기존 출발 API를 사용하도록 연결 지원
- 실패 원인 enum의 기본값과 설정 위치 점검
- 식량 실패가 `WagonBroken`으로 덮어쓰이지 않는지 점검
- 일반 거리 마모만으로 Traveling `WagonBroken` 실패가 나지 않는지 점검
- 식량 부족 속도 저하 로직·On/Off 플래그 계약 정리

### 완료 기준

- UI가 계산을 재구현하지 않고 Core 결과를 표시한다.
- 잘못된 구성은 동일한 이유로 차단된다.
- 정상 구성은 UI 버튼에서 출발한다.

### 선행 구현 메모

- **UI & Data:** Preparation 입력 UI와 View 연결 지점 필요
- **Framework:** 출발 성공 후 Traveling 라우팅 필요
- **Content & Tools:** 유효한 마차·동물·상품·식량 테스트 데이터 필요
- **Progression & System:** 기본 비용·수익 값이 동결되어야 정산 비교 가능

## 2026-07-15 — 실패 원인과 전체 루프 회귀

- UI 경로에서 정상 출발·과적 출발·출발 차단 테스트
- 식량 부족 유예 후 실패를 다시 검증
- 실패 원인을 식량 부족 관련 값으로 기록
- `WagonBroken`을 식량·일반 실패 기본값으로 사용하지 않도록 수정
- 식량 부족 속도 저하 플래그 On/Off 각각에서 유예·실패가 깨지지 않는지 확인
- 성공·실패 결과가 Settlement 데이터에 동일하게 전달되는지 확인
- Claim 후 새 Caravan 구성과 재출발 확인
- 무역 3회 반복 테스트 지원

### 완료 기준

- 성공과 실패가 모두 정확한 결과 데이터를 만든다.
- 실패 후 게임이 막히지 않는다.
- 실패 원인이 화면에 전달할 수 있는 상태다.

### 선행 구현 메모

- **Framework:** 경과 인게임 시간과 SettlementPending 전달
- **UI & Data:** 실패 사유와 정산 결과 표시 필드 연결
- **Content & Tools:** 식량 부족 재현용 긴 이동시간 프리셋
