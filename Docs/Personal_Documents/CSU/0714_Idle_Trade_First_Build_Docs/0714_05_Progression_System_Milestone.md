# 3. 현재 상태

- 기본 정산에서 Revenue, Cost, NetProfit 계산 가능
- 테스트 결과 Revenue 10, Cost 15, NetProfit -5와 재화 감소가 일치함
- 반복 무역에서 매회 동일한 손익이 누적됨
- 성장 계산기·지갑 API는 **코드에 존재**하나, 1차 **제출 경로·통과 조건에서는 사용하지 않음**
- 실패 원인 `WagonBroken` 문제는 경제 계산이 아니라 Core 결과 분류 문제로 우선 처리

# 4. 작업 일정

## 2026-07-14 — 기본 정산 계약 동결

- 1차 빌드에 실제 사용되는 비용 항목 목록 확정
- 판매 수익, 총비용, 순이익 출력 구조 확정
- UI 표시값과 재화 증감값의 기준 통일
- 성장·기부·투자·대출 보정이 **제출용 기본 정산**에 남아 있지 않은지 점검
- 과적 속도 보정과 식량 부족 속도 저하(플래그 On일 때)의 현재 상수 동결
- 마차 내구도/`WagonBroken`이 일반·식량 실패 비용·사유로 잘못 쓰이지 않는지 점검(전투 내구도 손실은 미구현)

### 완료 기준

- 같은 입력에서 같은 손익이 계산된다.
- UI가 표시할 정산 항목이 확정된다.
- 제외 기능의 보정이 기본 정산에 섞이지 않는다.

### 선행 구현 메모

- **Core Gameplay:** 실제 판매 결과와 실패 결과 제공
- **Content & Tools:** 상품 가격·식량 비용 데이터
- **UI & Data:** 정산 표시 항목 구조
- **Framework:** Claim 시 재화 저장 경로

## 2026-07-15 — 경제 회귀 테스트

- 성공 정산의 Revenue·Cost·NetProfit 검증
- 실패 정산의 비용·손실 검증
- Claim 전후 재화 변화 검증
- 동일 정산 중복 지급 여부 확인 지원
- 3회 반복 무역의 재화 누적 검증
- 음수 가격, 음수 비용, 정수 범위 이상 여부 확인
- 식량 부족 실패 후 재출발 가능한 최소 재화 상태 확인

### 완료 기준

- 정산 표시와 실제 재화가 일치한다.
- 반복 무역에서 재화가 예측 가능하게 변한다.
- 실패가 플레이를 복구 불가능하게 만들지 않는다.

### 선행 구현 메모

- **Framework:** Claim 중복 방지와 저장 완료
- **Core Gameplay:** 성공·실패 정산 데이터
- **UI & Data:** 표시값 비교 화면
