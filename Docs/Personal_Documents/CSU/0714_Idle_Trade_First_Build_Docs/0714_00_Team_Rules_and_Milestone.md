# 2. 축소된 1차 빌드 목표

```text
Title에서 새 게임 또는 이어하기
→ 플레이어 UI로 Caravan 준비
→ 출발 검증
→ Traveling 진행
→ 성공 또는 실패 정산
→ Claim
→ 다음 무역 준비
→ 저장 후 재실행 복구
```

성장 UX·성장 보정 적용, 기부, 투자, 대출, 튜토리얼, 계절·재난, 로드 이벤트 전체, 용병 **전투·고용 실기능**은 1차 빌드 통과 조건에서 제외한다.  
용병 **고용 UI 틀**은 제출 루프에 포함한다. 성장 관련 코드가 저장소에 있어도 제출 경로·통과 공식에 쓰지 않으면 실패로 보지 않는다.

# 3. 현재 프로젝트 상태

- Claim 및 중복 수령 방지

- 3회 연속 무역 루프
- 최대 적재량, 과적 허용, 견인 동물 혼합, 최소 동물 수 검증
- 식량 소모와 식량 부족 지속 후 실패 전환

- 디버그 도구를 대체할 플레이어용 Preparation UX
- 용병 고용 UI 단순 틀

- Title: SaveData 없을 때 Continue 비활성, Reset SaveData 확인 창

- 레이아웃·해상도·텍스트와 최종 통합

# 4. 팀 공통 우선순위

| 우선순위 | 의미 | 예시 |
|---|---|---|
| **P0** | 제출 불가 또는 플레이 불가 | 빌드 실패, 씬 진입 실패, 준비·출발 불가, 정산 불가 |
| **P1** | 핵심 결과 오류 | 저장 복구 실패, 중복 보상, 재화 오류, 실패 원인 오류 |
| **P2** | UX·표시 문제 | 잘못된 버튼 상태, 경고 누락, 텍스트·레이아웃 문제 |
| **P3** | 후속 개선 | 연출, 추가 콘텐츠, 상세 설명, 편의 기능 |

## 운영 원칙

- 7월 14일부터 신규 시스템을 추가하지 않는다.
- UX 연결을 위해 필요한 최소 API 보완만 허용한다.
- 공용 데이터 구조와 저장 구조의 대규모 변경을 금지한다.
- P0와 P1을 우선 수정하며 P3는 제출 후로 이관한다.
- 디버그 기능은 검증에 사용하되 제출 UX의 대체 수단으로 보지 않는다.
- Scene 수정은 **기존 SceneOwners**를 기본으로 한다. `InGame=UI / Title=Framework` 고정 분담은 1차 제출용 **일시 비상책**이었으며, 필요 시 **협의 후 일시적으로** 소유권자를 바꿀 수 있다. 같은 씬을 동시 수정하지 않는다.

# 5. 수정된 전체 마일스톤

## R0 — 2026-07-14: UX 연결 기준선

### 공통 목표

- 현재 동작하는 Core·Framework 기능을 플레이어 화면에 연결한다.
- Preparation, Traveling, Settlement의 입력·출력 계약을 동결한다.
- Title의 새 게임·이어하기 동작을 정리한다.
- 알려진 결함을 재현하고 담당자를 고정한다.

### 통과 조건

- 플레이어 UI에서 정상 Caravan을 구성할 수 있다.
- 출발 버튼이 실제 Core 검증 결과를 사용한다.
- 출발 성공 시 Traveling 화면으로 이동한다.
- 진행·정산 화면에 테스트 데이터가 아닌 현재 런타임 데이터가 표시된다.

## R1 — 2026-07-15: 전체 루프 통합과 회귀 테스트

### 공통 목표

- 새 게임과 이어하기 모두에서 전체 루프를 완료한다.
- 성공, 식량 부족 실패, Claim, 재출발을 UI에서 검증한다.
- Traveling과 SettlementPending의 재실행 복구를 검증한다.
- 실패 원인, 수익, 비용, 순이익 표시를 실제 결과와 일치시킨다.

### 통과 조건

- 디버그 Context Menu 없이 한 사이클을 완료한다.
- 성공·실패 후 다시 출발할 수 있다.
- 종료 후 이어하기가 올바른 화면과 데이터로 복구된다.
- 같은 정산이 두 번 지급되지 않는다.
- Console Error가 없다.

## R2 — 2026-07-16: RC와 제출

### 오전

- RC1 빌드
- 새 게임, 이어하기, 성공, 실패, 오프라인 완료, 정산 복구 테스트
- 목표 해상도와 입력 중복 테스트
- P0·P1 수정

### 오후

- RC2 또는 최종 RC 생성
- 최소 3명 독립 실행 확인
- 제출용 빌드, 소스 커밋, 문서 버전 확정
- 백업본 생성
- 23:59까지 제출

### 통과 조건

- 빌드가 독립 실행된다.
- 플레이어가 설명 없이 핵심 루프를 완료할 수 있다.
- Blocker가 없다.
- 재화 중복 지급과 저장 유실이 없다.
- 제출본과 소스 커밋을 추적할 수 있다.
